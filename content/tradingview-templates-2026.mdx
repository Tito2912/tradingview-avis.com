Les templates ne font pas “gagner des performances”. Ils font gagner **du temps** et de la **discipline**.

Un bon template réduit les décisions inutiles : vous voyez les mêmes repères, vous déclenchez les mêmes alertes, vous comparez les mêmes marchés.

<h2 id="pourquoi">Pourquoi les templates sont rentables</h2>

Sans templates, vous recréez :

- des paramètres d’indicateurs,
- des layouts multi‑graphiques,
- des alertes,
- des watchlists.

Résultat : erreurs, incohérences, et fatigue mentale.

<h2 id="layouts">Layouts : 1 objectif par écran</h2>

Le pattern le plus simple :

1. **Scan** : screeners + watchlist + 1 graphique propre.
2. **Execution** : multi‑timeframes + niveaux + plan.
3. **Review** : journaling / analyse après coup.

Chaque layout doit servir un objectif, sinon il devient du bruit.

<h2 id="indicateurs">Templates de graphiques (indicateurs)</h2>

Commencez par 2 templates :

- un template “tendance” (MA + structure),
- un template “momentum” (RSI/MACD + filtre).

Gardez vos paramètres stables, sinon vous ne savez plus ce que vous comparez.

<h2 id="alertes">Templates d’alertes : standardiser entrée / stop / sortie</h2>

Un template d’alertes efficace :

- suit un **naming** (actif + timeframe + signal),
- a un message court (action + contexte),
- limite le nombre d’alertes actives.

Votre alerte doit être lisible sur mobile, en 2 secondes.

<h2 id="checklist">Checklist d’organisation</h2>

- J’ai 2–3 layouts maximum, chacun avec un objectif.
- J’ai 2 templates d’indicateurs stables (tendance / momentum).
- J’ai un naming standard pour les alertes.
- J’ai un template d’alertes “entrée/stop/sortie”.
- Je supprime ce que je n’utilise pas (moins, c’est mieux).

Une fois cette base en place, TradingView devient un outil “système”, pas juste un graphique.
