<section className="post__section" id="pine">
<h2>Pine Script: Erstellen Sie eigene Indikatoren und Strategien</h2>
<p>
<strong>Pinienskript</strong> Die offizielle Dokumentation (Benutzerhandbuch und Sprachreferenz v6) ist gründlich und anfangsfreundlich und deckt immer noch fortgeschrittene Themen ab. Eine riesige Community-Bibliothek mit 150k+ Skripten (viele Open-Source-Funktionen) hilft Ihnen beim Beispiel zu lernen und Ideen zu remixen.
        </p>
<ul>
<li>Kombinieren Sie ein Breakout-Signal mit einem Trendfilter (z.B. steigendem MA200) und einem Volumenzustand.</li>
<li>Erstellen Sie eine einfache Long/short-Strategie mit Ein-/Ausgangsregeln und iterieren Sie über Backtests.</li>
<li>Exponate <em>Alarmbedingung()</em> löst aus, damit Ihr Skript serverseitige Warnungen und, falls erforderlich, Webhooks an externe Dienste ansteuern kann.</li>
</ul>
<p className="note">Scripts können mehrere Alarmauslöser aussetzen, aber <em>du</em> Alarme aus dem Diagramm UI erstellen/ aktivieren.</p>
</section>
<section className="post__section" id="alerts">
<h2>Serverseitige Warnungen: werden rund um die Uhr benachrichtigt, auch wenn die App geschlossen ist</h2>
<p>
          TradingView-Benachrichtigungen laufen auf TradingView’s <strong>Server</strong>, nicht Ihr Browser. Das bedeutet, Alarme können Sie feuern und benachrichtigen (Push, E-Mail, etc.), während Ihr Computer oder Telefon-App geschlossen ist. Sie können Alarme auf Preisschwellen, Indikator Kreuze oder Pine-basierte Bedingungen auslösen. <em>Webhoots</em> nachgeschaltete Dienstleistungen – testen Sie gründlich, bevor Sie sich auf automatisierte Aktionen verlassen.
        </p>

<section className="post__section" id="workflow">
<h2>Empfohlener Workflow: Anfänger → Zwischenprodukt → Fortgeschritten</h2>
<h3>1) Start sauber</h3>
<ul>
<li>Bauen Sie eine kurze <strong>Auf den Wunschzettel</strong> (10–20 Symbole) ausgerichtet auf Ihren Stil (Indizes, Sektoren, FX-Durs, Groß-Cap-Crypto).</li>
<li>Verwendung 1-3 <strong>Indikatoren</strong> max. zuerst (MA, RSI, Volume) um eine visuelle Überlastung zu vermeiden.</li>
<li>Speichern einer <strong>Layout</strong> pro Stil (z.B. täglich schwingen; Intraday 5/15/60).</li>
</ul>
<h3>2) Definieren Sie nützliche Warnungen</h3>
<ul>
<li>Schlüsselebenen (Ausbrüche von S/R, Rückzüge zu bewegenden Durchschnittswerten) mit <strong>Serverseitige Alarme</strong> so dass Sie offline angemeldet werden.</li>
<li>Geräuschreduzierung: Zustandsalarme auf einem Indikator (z.B. RSI verlässt eine Zone) oder auf Kerze schließen.</li>
</ul>
<h3>3) Benutzen Sie Bildschirme</h3>
<ul>
<li>Starten Sie mit 3–5 Filtern (Marktkappe, % Änderung, Volumen, MA Trend, RSI) und sparen Sie Ihre <strong>Voreinstellungen</strong>.</li>
<li>Vary-Zeitrahmen (H1, H4, D) um Multi-TF-Ausrichtung zu erkennen.</li>
</ul>
<h3>4) Graduate to Pine Script</h3>
<ul>
<li>Beginnen Sie, indem Sie ein bestehendes Community-Skript (eine Bedingung angeben, Eingaben aussetzen).</li>
<li>Exponate <em>Alarmbedingung()</em> Trigger und Test auf aktuelle Daten vor jeder Automatisierung.</li>
</ul>
<h3>5) Verify your data</h3>
<ul>
<li>Für Pro-Nutzung, abonnieren Sie das Recht <strong>offizielle Echtzeit-Bündel</strong> (z.B. US Markets) zur Vermeidung von Futterdiskursen.</li>
</ul>
