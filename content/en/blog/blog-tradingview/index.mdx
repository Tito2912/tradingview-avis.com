<section className="post__section" id="pine">
<h2>Pine Script: build your own indicators &amp; strategies</h2>
<p>
<strong>Pine Script</strong> is TradingView’s native language for building indicators and strategies you can backtest. The official documentation (User Manual and Language Reference v6) is thorough and beginner-friendly while still covering advanced topics. A huge community library with 150k+ scripts (many open-source) helps you learn by example and remix ideas.
        </p>
<ul>
<li>Combine a breakout signal with a trend filter (e.g., rising MA200) and a volume condition.</li>
<li>Create a simple long/short strategy with entry/exit rules and iterate via backtests.</li>
<li>Expose <em>alertcondition()</em> triggers so your script can drive server-side alerts and, if needed, webhooks to external services.</li>
</ul>
<p className="note">Scripts can expose multiple alert triggers, but <em>you</em> create/activate alerts from the chart UI.</p>
</section>
<section className="post__section" id="alerts">
<h2>Server-side alerts: get notified 24/7 even when the app is closed</h2>
<p>
          TradingView alerts run on TradingView’s <strong>servers</strong>, not your browser. That means alerts can fire and notify you (push, email, etc.) while your computer or phone app is closed. You can trigger alerts on price thresholds, indicator crosses, or Pine-based conditions. For automation, alerts can send <em>webhooks</em> to downstream services — test thoroughly before you rely on any automated actions.
        </p>

<section className="post__section" id="workflow">
<h2>Recommended workflow: beginner → intermediate → advanced</h2>
<h3>1) Start clean</h3>
<ul>
<li>Build a short <strong>watchlist</strong> (10–20 symbols) aligned with your style (indices, sectors, FX majors, large-cap crypto).</li>
<li>Use 1–3 <strong>indicators</strong> max at first (MA, RSI, Volume) to avoid visual overload.</li>
<li>Save a <strong>layout</strong> per style (e.g., swing daily; intraday 5/15/60).</li>
</ul>
<h3>2) Define useful alerts</h3>
<ul>
<li>Key levels (breakouts of S/R, pullbacks to moving averages) using <strong>server-side alerts</strong> so you’re notified offline.</li>
<li>Reduce noise: condition alerts on an indicator (e.g., RSI leaves a zone) or on candle close.</li>
</ul>
<h3>3) Use screeners</h3>
<ul>
<li>Start with 3–5 filters (market cap, % change, volume, MA trend, RSI) and save your <strong>presets</strong>.</li>
<li>Vary timeframes (H1, H4, D) to spot multi-TF alignment.</li>
</ul>
<h3>4) Graduate to Pine Script</h3>
<ul>
<li>Begin by tweaking an existing community script (add a condition, expose inputs).</li>
<li>Expose <em>alertcondition()</em> triggers and test on recent data before any automation.</li>
</ul>
<h3>5) Verify your data</h3>
<ul>
<li>For pro use, subscribe to the right <strong>official real-time bundles</strong> (e.g., US Markets) to avoid feed discrepancies.</li>
</ul>
