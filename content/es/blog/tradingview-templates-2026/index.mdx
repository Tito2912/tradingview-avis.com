Las plantillas no mejoran “rentabilidad”. Mejoran **consistencia**.

Consistencia = menos decisiones inútiles y menos errores.

<h2 id="why">Por qué las plantillas son rentables</h2>

Sin plantillas, rehaces constantemente:

- parámetros de indicadores,
- layouts multi-gráfico,
- alertas,
- watchlists.

Eso crea drift y fallos.

<h2 id="layouts">Layouts: un objetivo por pantalla</h2>

Patrón simple:

1. **Scan**: screener + watchlist + un gráfico limpio.
2. **Ejecución**: multi-timeframe + niveles + plan.
3. **Review**: journaling y post-análisis.

Cada layout debe tener una función.

<h2 id="charts">Plantillas de gráfico</h2>

Empieza con dos presets:

- “tendencia” (estructura + MA),
- “momentum” (RSI/MACD + filtro).

Mantén parámetros estables.

<h2 id="alerts">Plantillas de alertas: entrada / stop / salida</h2>

Una plantilla eficaz:

- naming estándar (activo + timeframe + señal),
- mensaje corto (móvil-friendly),
- pocas alertas activas.

<h2 id="checklist">Checklist de organización</h2>

- Tengo 2–3 layouts máximo, cada uno con objetivo.
- Tengo presets estables (tendencia/momentum).
- Uso un esquema de naming de alertas.
- Mantengo una plantilla core (entrada/stop/salida).
- Borro lo que no uso.

Con esta base, TradingView se vuelve un sistema, no solo gráficos.
