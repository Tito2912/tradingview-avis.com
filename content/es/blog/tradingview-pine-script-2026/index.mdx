Pine Script es potente cuando lo usas para **clarificar** un setup, no para perseguir complejidad.

Objetivo: señal legible, testeable y reproducible.

<h2 id="indicator-vs-strategy">Indicador vs estrategia</h2>

- Un **indicador** muestra estado (tendencia, momentum, niveles).
- Una **estrategia** formaliza entradas/salidas y permite backtesting.

Ruta recomendada: indicador → alertas → estrategia.

<h2 id="patterns">Patrones útiles (simples)</h2>

Patrones de inicio:

- señal principal (breakout),
- filtro de tendencia,
- filtro de calidad (volumen),
- trigger a cierre para reducir ruido,
- invalidación clara.

Coherencia > sofisticación.

<h2 id="backtesting">Backtesting: entiende qué pruebas</h2>

Sirve si:

- reglas explícitas,
- timeframe estable,
- aceptas sensibilidad a parámetros.

Evita sobre-optimizar: suele overfit.

<h2 id="alerts">Alertas: primer caso “producción”</h2>

Ejercicio ideal:

1. condición Pine,
2. alerta TradingView,
3. mensaje estable (símbolo, timeframe, señal),
4. webhook opcional.

Así pasas de script a workflow.

<h2 id="checklist">Checklist de calidad</h2>

- Señal explicable en una frase.
- Trigger claro (intrabar vs cierre).
- Evita comportamientos sorpresa (repaint accidental).
- Backtest con supuestos razonables.
- Alertas nombradas y probadas.

Con esto ya tienes un script “limpio”.
